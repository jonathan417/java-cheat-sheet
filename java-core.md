# Java Interview Cheat Sheet

## Core Java Concepts


| Concept          | Description                                                                                                                                                                                                                                                                                                                                                      | Example                                                                                                                                                                |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| JVM/JRE/JDK      | JVM: Executes bytecode.<br>• Java Virtual Machine<br>• Platform independent<br>• Runtime environment<br><br>JRE: JVM + libraries.<br>• Runtime only<br>• For running applications<br><br>JDK: JRE + development tools.<br>• For development<br>• Includes compiler                                                                                               | `javac Main.java` (JDK)<br>`java Main` (JVM)                                                                                                                           |
| Primitive Types  | Basic data types in Java.<br>• byte (1 byte): -128 to 127<br>• short (2 bytes): -32,768 to 32,767<br>• int (4 bytes): -2³¹ to 2³¹-1<br>• long (8 bytes): -2⁶³ to 2⁶³-1<br>• float (4 bytes): Single-precision IEEE 754<br>• double (8 bytes): Double-precision IEEE 754<br>• char (2 bytes): Unicode character<br>• boolean (1 bit, JVM uses 1 byte): true/false | `byte b = 10;`<br>`short s = 1000;`<br>`int i = 100000;`<br>`long l = 100000L;`<br>`float f = 1.5f;`<br>`double d = 1.5;`<br>`char c = 'A';`<br>`boolean done = true;` |
| Wrapper Classes  | Object representations of primitive types.<br>• Enable primitives in collections<br>• Provide utility methods<br>• Support null values                                                                                                                                                                                                                           | `Integer i = 10;`<br>`Character c = 'A';`<br>`Integer.parseInt("123");`                                                                                                |
| Access Modifiers | Control visibility of classes, methods, and fields.<br>• public: accessible everywhere<br>• protected: package + subclasses<br>• default (no modifier): package only<br>• private: class only                                                                                                                                                                    | `public void method() {}`<br>`protected int count;`<br>`String name;` // default<br>`private boolean flag;`                                                            |
| Static           | Belongs to class, not instances.<br>• Shared among all objects<br>• Accessed with class name<br>• Memory allocated once                                                                                                                                                                                                                                          | `public static final int MAX = 100;`<br>`public static void utility() {}`                                                                                              |
| Final            | Prevents modification.<br>• Classes: cannot be extended<br>• Methods: cannot be overridden<br>• Variables: constants                                                                                                                                                                                                                                             | `final class Immutable {}`<br>`final void noOverride() {}`<br>`final int CONSTANT = 42;`                                                                               |

## Object-Oriented Programming

| Concept | Description | Example |
|---------|-------------|---------|
| Encapsulation | Hiding data by making variables private + getters/setters.<br>• Restricts direct access to data<br>• Controls data validation<br>• Improves maintainability | `class User {`<br>&nbsp;&nbsp;`private String name;`<br>&nbsp;&nbsp;`public String getName() { return name; }`<br>&nbsp;&nbsp;`public void setName(String name) { this.name = name; }`<br>`}` |
| Inheritance | Subclass extends superclass.<br>• Reuses code<br>• Creates IS-A relationship<br>• Promotes code organization | `class Animal { void eat() {} }`<br>`class Dog extends Animal { void bark() {} }` |
| Polymorphism | Same interface, different implementations.<br>• Method overriding (runtime)<br>• Method overloading (compile-time)<br>• Enables flexibility | `// Overriding`<br>`class Shape { void draw() {} }`<br>`class Circle extends Shape { void draw() {} }`<br><br>`// Overloading`<br>`void print(String s) {}`<br>`void print(int i) {}` |
| Abstraction | Hide implementation details.<br>• Abstract classes & interfaces<br>• Focus on what not how<br>• Reduces complexity | `interface Drawable { void draw(); }`<br>`class Circle implements Drawable {`<br>&nbsp;&nbsp;`public void draw() { /*implementation*/ }`<br>`}` |
| Abstract Class | Partial implementation.<br>• Can have concrete methods<br>• Cannot be instantiated<br>• Can have constructor and fields | `abstract class Shape {`<br>&nbsp;&nbsp;`abstract void draw();`<br>&nbsp;&nbsp;`void move() { /*implementation*/ }`<br>`}` |
| Interface | Contract for implementing classes.<br>• Only abstract methods (prior to Java 8)<br>• Enables multiple inheritance<br>• Since Java 8: default and static methods | `interface Drawable { void draw(); }`<br>`interface Colorable { void setColor(String c); }`<br>`class Circle implements Drawable, Colorable {}` |

## Collections Framework

| Type | Implementations | Characteristics | Example |
|------|----------------|-----------------|---------|
| List | ArrayList | Fast access, slow insert/delete.<br>• Array-based implementation<br>• Random access in O(1)<br>• Dynamically resizable | `List<String> list = new ArrayList<>();`<br>`list.add("item"); list.get(0);` |
| | LinkedList | Fast insert/delete, slow access.<br>• Doubly-linked list implementation<br>• O(1) operations at both ends<br>• Implements Deque interface | `LinkedList<String> list = new LinkedList<>();`<br>`list.addFirst("item"); list.removeLast();` |
| | Vector | Synchronized, thread-safe, slower than ArrayList.<br>• Legacy class (pre-Collections framework)<br>• All methods synchronized<br>• Similar to ArrayList but thread-safe | `Vector<String> vector = new Vector<>();`<br>`vector.add("item"); vector.get(0);` |
| | Stack | LIFO (Last-In-First-Out), extends Vector.<br>• Legacy class<br>• All operations synchronized<br>• Consider ArrayDeque instead | `Stack<String> stack = new Stack<>();`<br>`stack.push("item"); stack.pop(); stack.peek();` |
| Set | HashSet | Fast operations, unordered.<br>• Hash table implementation<br>• No duplicates allowed<br>• O(1) for add/remove/contains | `Set<Integer> set = new HashSet<>();`<br>`set.add(1); set.contains(1);` |
| | TreeSet | Ordered, slower operations.<br>• Red-black tree implementation<br>• Elements sorted naturally or by comparator<br>• O(log n) for add/remove/contains | `TreeSet<String> set = new TreeSet<>();`<br>`set.add("b"); set.add("a");` // stores in order |
| | LinkedHashSet | Ordered by insertion, fast operations.<br>• HashSet with linked list<br>• Preserves insertion order<br>• Slightly slower than HashSet | `LinkedHashSet<String> set = new LinkedHashSet<>();`<br>`set.add("a"); set.add("b");` // keeps insertion order |
| Map | HashMap | Fast operations, unordered.<br>• Hash table implementation<br>• O(1) for put/get/remove<br>• Allows one null key and multiple null values | `Map<String, Integer> map = new HashMap<>();`<br>`map.put("key", 1); map.get("key");` |
| | TreeMap | Ordered by key, slower operations.<br>• Red-black tree implementation<br>• Keys sorted naturally or by comparator<br>• O(log n) for put/get/remove | `TreeMap<String, Integer> map = new TreeMap<>();`<br>`map.put("b", 2); map.put("a", 1);` // keys sorted |
| | LinkedHashMap | Ordered by insertion, fast operations.<br>• HashMap with linked list<br>• Preserves insertion order<br>• Slightly slower than HashMap | `LinkedHashMap<String, Integer> map = new LinkedHashMap<>();`<br>`map.put("a", 1); map.put("b", 2);` // insertion order |
| | ConcurrentHashMap | Thread-safe operations, segments for concurrency.<br>• Thread-safe without full synchronization<br>• High concurrency with segments<br>• No null keys or values allowed | `ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();`<br>`map.put("key", 1); // Thread-safe operations` |
| Queue | PriorityQueue | Ordered by priority.<br>• Heap-based implementation<br>• Elements ordered by natural ordering or comparator<br>• Not thread-safe | `PriorityQueue<Integer> pq = new PriorityQueue<>();`<br>`pq.add(3); pq.add(1); pq.poll();` // returns 1 |
| | ArrayDeque | Double-ended queue.<br>• Array-based implementation<br>• Fast operations at both ends<br>• More efficient than Stack or LinkedList | `ArrayDeque<String> deque = new ArrayDeque<>();`<br>`deque.addFirst("a"); deque.addLast("b");` |
| | ConcurrentLinkedQueue | Thread-safe, non-blocking queue.<br>• Lock-free implementation<br>• High-throughput, low-latency<br>• Good for producer-consumer patterns | `ConcurrentLinkedQueue<String> queue = new ConcurrentLinkedQueue<>();`<br>`queue.add("item"); queue.poll();` |
| Concurrent Collections | CopyOnWriteArrayList | Thread-safe, immutable snapshots for iteration.<br>• Thread-safe for reading<br>• Expensive for frequent modifications<br>• Snapshot semantics for iterators | `CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();`<br>`list.add("item"); // Thread-safe for reading` |
| | CopyOnWriteArraySet | Thread-safe, uses CopyOnWriteArrayList.<br>• Set interface<br>• No duplicates<br>• Snapshot semantics for iterators | `CopyOnWriteArraySet<String> set = new CopyOnWriteArraySet<>();`<br>`set.add("item"); // Safe for concurrent access` |
| | BlockingQueue | Thread coordination, supports waiting.<br>• Blocks if queue full/empty<br>• Supports producer-consumer pattern<br>• Various implementations available | `BlockingQueue<Task> queue = new LinkedBlockingQueue<>(10);`<br>`queue.put(task); // Blocks if queue full`<br>`Task task = queue.take(); // Blocks if queue empty` |

## Exception Handling

| Concept | Description | Example |
|---------|-------------|---------|
| Checked Exceptions | Must be caught/declared; compile-time.<br>• Verified by compiler<br>• Extends Exception but not RuntimeException<br>• Examples: IOException, SQLException | `void readFile() throws IOException {`<br>&nbsp;&nbsp;`FileReader fr = new FileReader("file.txt");`<br>`}` |
| Unchecked Exceptions | Runtime exceptions; no need to catch.<br>• Not verified by compiler<br>• Extends RuntimeException<br>• Examples: NullPointerException, ArrayIndexOutOfBoundsException | `int divide(int a, int b) {`<br>&nbsp;&nbsp;`return a / b; // throws ArithmeticException if b is 0`<br>`}` |
| try-catch-finally | try: code that might throw exception.<br>• catch: handle exception<br>• finally: always executes<br>• Multiple catch blocks possible | `try {`<br>&nbsp;&nbsp;`int result = 10 / 0;`<br>`} catch (ArithmeticException e) {`<br>&nbsp;&nbsp;`System.out.println("Cannot divide by zero");`<br>`} finally {`<br>&nbsp;&nbsp;`System.out.println("Always executes");`<br>`}` |
| try-with-resources | Auto-closes resources implementing AutoCloseable.<br>• Resources closed in reverse order of creation<br>• Introduced in Java 7<br>• Clean alternative to finally block | `try (FileReader fr = new FileReader("file.txt");`<br>&nbsp;&nbsp;&nbsp;&nbsp; `BufferedReader br = new BufferedReader(fr)) {`<br>&nbsp;&nbsp;`String line = br.readLine();`<br>`} // fr and br auto-closed` |
| throw/throws | throw: throws exception.<br>• throws: declares exception in method signature<br>• throw creates a new exception instance<br>• throws lists potential exceptions | `void validate(int age) {`<br>&nbsp;&nbsp;`if (age < 0) throw new IllegalArgumentException("Age can't be negative");`<br>`}`<br>`void readFile() throws IOException { /* implementation */ }` |

## Multithreading

| Concept | Description | Example |
|---------|-------------|---------|
| Thread Creation | Two ways to create threads in Java.<br>• Extend Thread class<br>• Implement Runnable interface<br>• Runnable is preferred (no multiple inheritance issues) | `// Extending Thread`<br>`class MyThread extends Thread {`<br>&nbsp;&nbsp;`public void run() { System.out.println("Thread running"); }`<br>`}`<br><br>`// Implementing Runnable`<br>`class MyRunnable implements Runnable {`<br>&nbsp;&nbsp;`public void run() { System.out.println("Runnable running"); }`<br>`}`<br>`Thread t = new Thread(new MyRunnable());`<br>`t.start();` |
| Synchronization | Mechanism to control thread access to shared resources.<br>• Prevents race conditions<br>• Uses intrinsic locks (monitors)<br>• Can be applied to methods or blocks | `// Method synchronization`<br>`synchronized void increment() { count++; }`<br><br>`// Block synchronization`<br>`synchronized(this) { count++; }` |
| Volatile | Ensures variable visibility across threads.<br>• Prevents CPU caching issues<br>• No atomicity guarantee<br>• Good for flags and status variables | `private volatile boolean flag = false;`<br>`// Ensures visibility across threads` |
| Atomic Classes | Thread-safe operations on single variables.<br>• Lock-free operations<br>• Uses compare-and-swap (CAS)<br>• Better performance than synchronized | `AtomicInteger counter = new AtomicInteger(0);`<br>`counter.incrementAndGet(); // Thread-safe increment` |
| Thread States | Lifecycle states of a thread.<br>• New: created but not started<br>• Runnable: executing or ready to execute<br>• Blocked: waiting for monitor lock<br>• Waiting: waiting indefinitely<br>• Timed Waiting: waiting for specified time<br>• Terminated: completed execution | `Thread t = new Thread(); // NEW`<br>`t.start();              // RUNNABLE`<br>`t.sleep(1000);          // TIMED_WAITING`<br>`t.join();               // WAITING`<br>`synchronized(obj) {}    // potentially BLOCKED` |
| Executor Framework | Manages thread pools and task execution.<br>• Decouples task submission from execution<br>• Reuses threads<br>• Various pool implementations<br>• Better than creating threads manually | `ExecutorService executor = Executors.newFixedThreadPool(10);`<br>`executor.submit(() -> System.out.println("Task executed"));`<br>`executor.shutdown();` |
| Future/CompletableFuture | Handle asynchronous computations.<br>• Future: basic async result container<br>• CompletableFuture: composable async tasks<br>• Supports callback chaining<br>• Introduced in Java 8 | `// Future`<br>`Future<String> future = executor.submit(() -> "Result");`<br>`String result = future.get(); // Blocks until complete`<br><br>`// CompletableFuture`<br>`CompletableFuture<String> cf = CompletableFuture.supplyAsync(() -> "Result");`<br>`cf.thenAccept(System.out::println);` |

## Spring Framework

| Concept | Description | Example |
|---------|-------------|---------|
| IoC Container | Manages object creation and lifecycle.<br>• Inversion of Control principle<br>• Creates and wires objects<br>• BeanFactory and ApplicationContext implementations | `@Configuration`<br>`public class AppConfig {`<br>&nbsp;&nbsp;`@Bean`<br>&nbsp;&nbsp;`public UserService userService() {`<br>&nbsp;&nbsp;&nbsp;&nbsp;`return new UserServiceImpl();`<br>&nbsp;&nbsp;`}`<br>`}`<br>`// Usage`<br>`ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);`<br>`UserService userService = context.getBean(UserService.class);` |
| Dependency Injection | Objects receive dependencies instead of creating them.<br>• Loose coupling between components<br>• Constructor, setter, and field injection<br>• Easier testing with mock objects | `// Constructor injection`<br>`@Service`<br>`public class UserServiceImpl implements UserService {`<br>&nbsp;&nbsp;`private final UserRepository userRepository;`<br>&nbsp;&nbsp;`@Autowired`<br>&nbsp;&nbsp;`public UserServiceImpl(UserRepository userRepository) {`<br>&nbsp;&nbsp;&nbsp;&nbsp;`this.userRepository = userRepository;`<br>&nbsp;&nbsp;`}`<br>`}`<br><br>`// Field injection`<br>`@Service`<br>`public class OrderService {`<br>&nbsp;&nbsp;`@Autowired`<br>&nbsp;&nbsp;`private UserService userService;`<br>`}` |
| Bean Scopes | Controls the lifecycle and visibility of beans.<br>• Singleton (default): one instance per container<br>• Prototype: new instance each time requested<br>• Request: one instance per HTTP request<br>• Session: one instance per HTTP session | `@Component`<br>`@Scope("singleton") // Default`<br>`public class SingletonBean {}`<br><br>`@Component`<br>`@Scope("prototype")`<br>`public class PrototypeBean {}`<br><br>`@Component`<br>`@Scope(value = WebApplicationContext.SCOPE_REQUEST, proxyMode = ScopedProxyMode.TARGET_CLASS)`<br>`public class RequestScopedBean {}` |
| AOP | Aspect-Oriented Programming.<br>• Separate cross-cutting concerns<br>• Method interception with pointcuts<br>• Common uses: logging, security, transactions | `@Aspect`<br>`@Component`<br>`public class LoggingAspect {`<br>&nbsp;&nbsp;`@Before("execution(* com.example.service.*.*(..))")`<br>&nbsp;&nbsp;`public void logBefore(JoinPoint joinPoint) {`<br>&nbsp;&nbsp;&nbsp;&nbsp;`System.out.println("Before " + joinPoint.getSignature().getName());`<br>&nbsp;&nbsp;`}`<br>&nbsp;&nbsp;`@Around("@annotation(Loggable)")`<br>&nbsp;&nbsp;`public Object logExecution(ProceedingJoinPoint joinPoint) throws Throwable {`<br>&nbsp;&nbsp;&nbsp;&nbsp;`long start = System.currentTimeMillis();`<br>&nbsp;&nbsp;&nbsp;&nbsp;`Object result = joinPoint.proceed();`<br>&nbsp;&nbsp;&nbsp;&nbsp;`long time = System.currentTimeMillis() - start;`<br>&nbsp;&nbsp;&nbsp;&nbsp;`System.out.println("Execution time: " + time + "ms");`<br>&nbsp;&nbsp;&nbsp;&nbsp;`return result;`<br>&nbsp;&nbsp;`}`<br>`}` |
| Annotations | Declarative configuration.<br>• @Component: generic Spring-managed component<br>• @Service: business logic layer<br>• @Repository: data access layer<br>• @Controller: web controller<br>• @Autowired: dependency injection | `@Component // Generic component`<br>`public class Component {}`<br><br>`@Service // Business logic`<br>`public class UserService {}`<br><br>`@Repository // Data access`<br>`public class UserRepository {}`<br><br>`@Controller // Web controller`<br>`public class UserController {}` |
| Spring Boot | Convention over configuration.<br>• Auto-configuration<br>• Standalone applications<br>• Embedded servers<br>• Production-ready features | `@SpringBootApplication`<br>`public class Application {`<br>&nbsp;&nbsp;`public static void main(String[] args) {`<br>&nbsp;&nbsp;&nbsp;&nbsp;`SpringApplication.run(Application.class, args);`<br>&nbsp;&nbsp;`}`<br>`}`<br><br>`// application.properties`<br>`spring.datasource.url=jdbc:mysql://localhost:3306/db`<br>`spring.jpa.hibernate.ddl-auto=update` |
| Spring MVC | Web application framework.<br>• Model-View-Controller pattern<br>• RESTful API support<br>• Request mapping annotations<br>• Data binding | `@Controller`<br>`@RequestMapping("/users")`<br>`public class UserController {`<br>&nbsp;&nbsp;`@Autowired`<br>&nbsp;&nbsp;`private UserService userService;`<br>&nbsp;&nbsp;`@GetMapping("/{id}")`<br>&nbsp;&nbsp;`public String getUser(@PathVariable Long id, Model model) {`<br>&nbsp;&nbsp;&nbsp;&nbsp;`model.addAttribute("user", userService.findById(id));`<br>&nbsp;&nbsp;&nbsp;&nbsp;`return "user/details"; // view name`<br>&nbsp;&nbsp;`}`<br>&nbsp;&nbsp;`@PostMapping`<br>&nbsp;&nbsp;`public ResponseEntity<User> createUser(@RequestBody User user) {`<br>&nbsp;&nbsp;&nbsp;&nbsp;`User saved = userService.save(user);`<br>&nbsp;&nbsp;&nbsp;&nbsp;`return ResponseEntity.created(URI.create("/users/" + saved.getId()))`<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;`.body(saved);`<br>&nbsp;&nbsp;`}`<br>`}` |

## SQL/Database

| Concept | Description | Example |
|---------|-------------|---------|
| JDBC | Java Database Connectivity.<br>• Standard API for database access<br>• Driver-based architecture<br>• Low-level database operations | `Connection conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/db", "user", "password");`<br>`Statement stmt = conn.createStatement();`<br>`ResultSet rs = stmt.executeQuery("SELECT * FROM users");`<br>`while (rs.next()) {`<br>`  String name = rs.getString("name");`<br>`}` |
| Connection Pooling | Reuse database connections.<br>• Improves performance<br>• Manages connection lifecycle<br>• Popular implementations: HikariCP, C3P0 | `// HikariCP example`<br>`HikariConfig config = new HikariConfig();`<br>`config.setJdbcUrl("jdbc:mysql://localhost:3306/db");`<br>`config.setUsername("user");`<br>`config.setPassword("password");`<br>`HikariDataSource dataSource = new HikariDataSource(config);`<br>`Connection conn = dataSource.getConnection();` |
| JPA/Hibernate | Object-Relational Mapping (ORM).<br>• Maps Java objects to database tables<br>• Reduces boilerplate SQL code<br>• Provides caching and lazy loading | `@Entity`<br>`@Table(name = "users")`<br>`public class User {`<br>`  @Id`<br>`  @GeneratedValue(strategy = GenerationType.IDENTITY)`<br>`  private Long id;`<br>`  `<br>`  @Column(nullable = false)`<br>`  private String name;`<br>`  `<br>`  @OneToMany(mappedBy = "user", cascade = CascadeType.ALL)`<br>`  private List<Order> orders;`<br>`}` |
| Transaction | Logical unit of work.<br>• ACID properties (Atomicity, Consistency, Isolation, Durability)<br>• Declarative with @Transactional in Spring<br>• Rollback on exceptions | `@Service`<br>`public class UserService {`<br>`  @Transactional`<br>`  public void transferMoney(Long fromId, Long toId, BigDecimal amount) {`<br>`    Account from = accountRepo.findById(fromId).get();`<br>`    Account to = accountRepo.findById(toId).get();`<br>`    from.debit(amount);`<br>`    to.credit(amount);`<br>`    accountRepo.save(from);`<br>`    accountRepo.save(to);`<br>`  }`<br>`}` |
| Query Types | Different categories of SQL statements.<br>• DDL: Define structure (CREATE, ALTER, DROP)<br>• DML: Manipulate data (INSERT, UPDATE, DELETE)<br>• DCL: Control access (GRANT, REVOKE)<br>• TCL: Transaction control (COMMIT, ROLLBACK) | `-- DDL (Data Definition Language)`<br>`CREATE TABLE users (id INT PRIMARY KEY, name VARCHAR(100));`<br>`ALTER TABLE users ADD COLUMN email VARCHAR(100);`<br><br>`-- DML (Data Manipulation Language)`<br>`INSERT INTO users (id, name) VALUES (1, 'John');`<br>`UPDATE users SET name = 'Jane' WHERE id = 1;`<br>`DELETE FROM users WHERE id = 1;`<br><br>`-- DCL (Data Control Language)`<br>`GRANT SELECT ON users TO user;`<br>`REVOKE SELECT ON users FROM user;`<br><br>`-- TCL (Transaction Control Language)`<br>`START TRANSACTION;`<br>`INSERT INTO users (id, name) VALUES (1, 'John');`<br>`COMMIT;`<br>`ROLLBACK;` |
| Normalization | Database design technique to reduce redundancy.<br>• 1NF: Atomic values (no repeating groups)<br>• 2NF: 1NF + No partial dependencies<br>• 3NF: 2NF + No transitive dependencies | `-- 1NF: Atomic values (no repeating groups)`<br>`users(id, name, email)`<br><br>`-- 2NF: 1NF + No partial dependencies`<br>`users(id, name, email)`<br>`orders(id, user_id, product, amount)`<br><br>`-- 3NF: 2NF + No transitive dependencies`<br>`users(id, name, email)`<br>`orders(id, user_id, product_id, amount)`<br>`products(id, name, price)` |
| Indexes | Improve query performance.<br>• Speed up data retrieval<br>• Added automatically for primary/foreign keys<br>• Trade-off: faster reads, slower writes | `-- Primary key (automatically indexed)`<br>`CREATE TABLE users (`<br>`  id INT PRIMARY KEY,`<br>`  name VARCHAR(100)`<br>`);`<br><br>`-- Adding an index`<br>`CREATE INDEX idx_user_name ON users(name);`<br><br>`-- Composite index`<br>`CREATE INDEX idx_user_name_email ON users(name, email);` |

## Testing
